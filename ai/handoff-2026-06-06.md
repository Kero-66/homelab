## Completed Work

### 1. FileFlows — YouTube flows failing (commit `23c34f1` → `9e7a5f6`)

**Problem**: All YouTube URL processing failed instantly with `ProcessingFailed`.

**Root causes** (two separate issues, both fixed):
- Plugin version mismatch: container pinned to `revenz/fileflows:26.03`, plugins auto-updated to `26.04`. Runner rejected plugins with "Invalid Version - won't use" → `FileFlows.Web.FlowElements.InputUrl` failed to load.
- Missing volume mount: flow writes to `/data/youtube` inside container, but that path was never mounted.
- On remount, `/mnt/Data/media/youtube` was owned by root → `chown -R 1000:1000` applied.

**Changes**: `truenas/stacks/fileflows/compose.yaml` — image bumped to `26.04`, added `/mnt/Data/media/youtube:/data/youtube` volume.

**Documented in**: `.github/TROUBLESHOOTING.md` — new "FileFlows" section with both failure modes.

**Prevention note**: Do not pin FileFlows to a minor version tag — plugins auto-update and the runner rejects version mismatches.

### 3. Cleanuparr — permissions errors on downloads (commit `2db271d`)

**Problem**: `[DownloadCleaner] skip | file does not exist or insufficient permissions | /data/downloads/qbittorrent/completed/...` — repeating every 3 hours.

**Root cause**: Cleanuparr's unlinked download feature checks hardlinks directly on the filesystem (per official docs: "mount the downloads directory the same way it is mounted for the download client"). The compose only mounted `/mnt/Data/media:/data` — no downloads mount. Also, the repo had post-dockhand network names (`arr-stack_default`, `jellyfin_default`) but TrueNAS uses `ix-` prefixed names pre-cutover.

**Changes**: `truenas/stacks/arr-stack/compose.yaml`:
- Added `/mnt/Data/downloads:/data/downloads` to cleanuparr volumes
- Fixed network names: `ix-arr-stack_default`, `ix-jellyfin_default` (with comments marking cutover points)

**Note**: `config.yml` in `/mnt/Fast/docker/cleanuparr/` is stale/unused — actual config lives in `cleanuparr.db` (SQLite). `dry_run` is `0` (off) in the database.
