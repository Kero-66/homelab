## Pending: Prowlarr Provider Setup

The proper way to add Prowlarr (and other providers) is via the Mylar3-compatible API:

```
POST /api?cmd=addProvider
  &providertype=newznab   # or torznab
  &name=Prowlarr
  &host=http://prowlarr:9696
  &prov_apikey=<PROWLARR_API_KEY>
  &enabled=true
  &apikey=<COMICARR_API_KEY>
```

**Blocker**: `api_key = None` in `config.ini` even though the user can see an API key in the UI (Settings → General → API Key). The key was likely written to a location not reflected in the file we're reading, or was generated after the last restart without persisting to disk.

**Next step**: 
1. Get the comicarr API key — either from the UI directly or by re-reading config.ini after confirming which file the running process is actually using
2. Use it to call `addProvider` for both `newznab` and `torznab` pointing at Prowlarr (`http://prowlarr:9696`)
3. Verify via the UI that providers appear under Settings → Search Providers

**Also check**: Whether the `torznab_*` fields we wrote to config.ini actually work, or whether providers must be added via the API to appear in the UI.
